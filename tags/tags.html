Теги HTML 5 и CSS 3

  • background-color - цвет фона
  • background-image -
    body {
    background-image: url
    (http://сайт_точка_ру/фон.jpg)
    }     - Задать фон-картинку для сайта.
    Доп. параметры фона:
    • background-repeat - Дублирование фонового изображения.
      Значения:
      • repeat - Дублирование по вертикали и горизонтали.
      • repeat-x - Только в горизонтали.
      • repeat-y - Только в вертикали.
      • space - Дублируется, пока не заполнит всю область.
      • round - Дублируется, чтобы в области поместилось
        целое число рисунков.
        Если это не удается, меняются размеры фонового рисунка.
      • no-repeat - Не дублируется.
    • background-attachment - Прокрутка фона при прокрутке документа.
      Значения:
      • fixed - Зафиксировано.
      • scroll - Прокручивается(по умолчанию).
    • background-position - Положение изображение в рамках док-та.
      Т.е., расположение первого экземпляра картинки.
      Именно от него будет производиться дублирование в разные стороны.
      Варианты:
      • В процентах, от левого верхнего угла.
        Первое смещение вправо, второе - вниз.
      • В единицах длины(пиксели).
      • Ключевыми словами: center, top, right, bottom, left.
      • Если указано только одно свойство(в процентах или длине),
        то это принимается как значение по горизонтали.
        По вертикали рисунок выравнивается на 50% по умолчанию.
      • Можно комбинировать значения "%" и "px".
    • background-origin - Область позиционирования.
    • background-size - Задает масштаб.
    • background-clip - Как фон будет выводиться под границами.
  • color - цвет текста
  • margin - внешний отступ, поле.
    Порядок отступов:
    Верх-право-низ-лево.
    Пример: p{margin: 10px 15px 10px 25px}
  • float(right, left) - плавающий элемент
  • padding - внутренний отступ (до границы)
  • border-width - толщина границы блочного элемента.
    Значения: (пиксели)(thin, medium, thik)
  • border-color - Цвет границ.
  • border-style - оформление(стиль) границы.
    Значения:
    • solid-сплошная
    • dotted-точечная
    • double-двойная
    • dashed-пунктир
    • groove-эффект углубления
    • inset-вкладка вдавленная в страницу
    • onset-выступающая из страницы
    • ridge-выступающая кромка
  • border-top, -right, -bottom, -left -
    Задать сразу ширину, тип линии и цвет в одном правиле.
    Или просто border, для всех границ сразу.
  • Углы в CSS.
    Значения:
    • deg - Градусы.
    • rad - Радианы
    • grad - Грады.
    • turn - Поворот круга.
  • id - Идентификатор, присваиваемый html-тегам,
    для изменения стилей, отдельно для каждого элемента с id.
  • Псевдокласс visited - посещенные ссылки.
    A:visited{color:red} - цвет посещённой ссылки.
  • Псевдокласс link - непосещённые ссылки.
  • Псевдокласс active - активные ссылки.
  • Псевдокласс focus - Ссылки, с фокусом на них.
  • Псевдокласс hover - Ссылки, над видимым содержимым
    которых находится курсор.
  • text-indent - Отступ первого слова в строке(красная строка). %, px.
  • text-align - Выравнивание текста.
    Варианты:
    • left - По левому краю.
    • right - По правому краю.
    • center - По центру.
    • justify - По ширине.
    • start - По началу текста, независимо, от написания.
      Справа налево или слева направо.
    • end - По концу текста.
  • text-decoration - Подчёркивание, зачёркивание и надчёркивание текста.
    Значения:
    • underline - Обычное подчёркивание.
    • overline - Черта сверху.
    • line-through - Зачёркнутый текст.
    • nont - Нет ничего. По умолчанию.
  • letter-spacing - Интервал между буквами.
  • word-spacing - Интервал между словами.
  • line-height - Межстрочный интервал.
  • text-transform - Изменение регистра букв.
    Значения:
    • capitalize - Первая буква каждого слова заглавная.
    • uppercase - Все буквы заглавные.
    • lowercase - Все буквы строчные.
    • no - Нет ничего.
  • columns - Создание многоколоночного текста.
    Дополнительные свойства:
    • column-fill -
      Со значением auto - заполняет колонки последовательно.
      Со значением balance - равномерно.
    • column-gap - Расстояние между колонками.
    • column-rule - Граница между колонок.
      Подробнее:
      • column-rule-width - Ширина границы.
      • column-rule-style - Стиль линии.
      • column-rule-color - Цвет.
    • column-width - Ширина колонки.
  • font-family - Тип шрифта.
    Самые распространенные: serif, sans-serif,
    monospace, curcive, fantasy.
  • font-style - Стиль шрифта.
    • normal - Обычное отображение.
    • italic - Курсив.
    • oblique - Наклонный.
  • font-size - Размер шрифта.
  • font-variant - Стиль шрифта, при котором все буквы выглядят
    как заглавные, но немного меньшего размера
    чем реальные заглавные. Значения: small-caps - вкл.
    normal - выкл.(по умолчанию.)
  • font-weight - Жирность текста. Значения имеют девять градаций:
    От 100 до 900, но не все шрифты поддерживают эти градации.
    Также есть значения normal(обычный(100)) и bold(700).
  • font - Сокращение написания всех свойств текста сразу.
    Подробности:
    • Порядок указания свойств:
      1. font-style
      2. font-variant
      3. font-weight
      4. font-size
      5. line-height
      6. font-family
    • Каждое из свойств может не задаваться,
      тогда будет значение по умолчанию.
    • font-size и line-height задаются через слэш "/".
    • Названия шрифтов перечисляются через запятую.
    • Пример: body{italicbold 12pt/14pt Arial serif}
  • @font-face - Загрузку авторских шрифтов.(@ Эй-ти)
    Если таких шрифтов нет на компьютере пользователя,
    они подгружаются с сервера. Пример:
    @font-face {font-family: My Font; src url (http://ссылка_на_шрифт)}
    После такого правила, можно указывать название этого шрифта в свойствах.
  • position - Задает местоположение html-блокам.
    Значения:
    • static - Обычное положение блока. Статическая схема.
    • relative - Относительная схема.
      Блок распологается как обычный, в потоке.
      Затем он может быть смещён, и другие блоки,
      никак на это реагировать не будут.
      Блок перекрывает любые другие блоки нормального потока.
      Может быть перекрыт такими же блоками (или абсолютными),
      заданными позднее.
    • absolute - Абсолютная схема.
      Свойства абсолютного блока:
      • Позиция указывается относительно ближайшего
        контейнера-родителя с абсолютной или
        относительной схемой.
      • Если такого⇑ родителя не имеется,
        то позиция будет относительно начального контейнера.
      • Может перекрывать блок, расположенный ранее,
        и перекрываться блоками, расположенными позже.
      • Т.е., такой блок может быть перекрыт только
        другим абсолютным или относительным блоком,
        заданным после него.
      • Поля абсолютных главных блоков не перекрываются
        никакими другими полями.
    • fixed - Фиксированная схема.
    • sticky - Сочетание относительной и фиксированной схем.
  • z-index - Расположение перекрывающих блоков, относительно друг друга.
    Или, кто за кем прячется.
    Значение по умолчанию "auto". Значение "1" перекроет блок с "auto",
    и чем больше это число тем выше приоритет блока относительно других.
  • overflow - Способ обработки переполненного блока.
    Значения:
    • visible - По умолчанию.
      Отображение всей содержимой в блоке информации.
    • hidden - Усечение лишней информации
      (не будет отображаться).
    • scroll - При переполнении выводится
      полоса прокрутки. Размеры блока остаются прежними.
    • auto - Браузер решает как быть.
  • visibility - Видимость/невидимость блока.
    visible - Видимый.
    hidden - Невидимый. Все блоки ведут себя так, будто его нет.
  • cursor - Авторский вид курсора.
    Пример:
    • body{cursor url ("myfirst.cur"), url ("mysecond.cur")}
      Можно задавать несколько вариантов, в случае если
      первый не подгрузится, будет использован следующий
      в списке вариант курсора.
      Изменять форму можно индивидуально
      для каждого элемента страницы.
    Ключевые слова:
    • default - Курсор по умлочанию.
    • crosshair - Крестик.
    • text - Специальный вид для текста,
      который надо выделить.
    • wait - Песочные часы.
    • auto - Автоматически задается браузером,
      в зависимости от контекста.
  • Фильтры. Задаются атрибутом filter.
  • blur - Размытие информации в определенном направлении
    и с определенной интенсивностью. Пример:
    img {filter: blur(50)}
    - число, это интенсивность размытия изображения.
  • opacity - Прозрачность. Задаётся в процентах.
  • drop-shadow - Создается тень-копия.
    Параметры:
    • Сдвиг по х - Горизонтальное смещение
      относительно исходника.
    • Сдвиг по у - Вертикальное смещение
      относительно исходника.
    • Размытие - Радиус размытия тени.
    • Цвет - Цвет копии. По умолч-ю черная.
    • Пример:
      img {width:400; filter:drop-shadow(120 120 120px yellow)}
  • grayscale - Преобразование цветного объекта в ч/б.
    Задается в процентах.
  • invert - Инверсия цветов(в процентах).
  • saturate - Насыщенность.
  • sepia - Сепия.
  • brigthness - Яркость.
  • contrast - Контраст.
  • ‹ol› - нумерованный список
    Атрибут type и его значения:
    • type=1 - Арабские цифры(по умолчанию)
    • type=A - Прописные латинские буквы.
    • type=a - Строчные латинские буквы.
    • type=I - Большие римские цифры.
    • type=i - Маленькие латинские цифры.
  • Свойства ol и ul:
    • list-style-type - Задает тип маркера или нумерацию.
      Выбирается из стандартных маркеров CSS3.
      Значения(ключевые слова):
      • none - Никакого маркера вообще.
      • disk - Закрашенный кружок.
      • circle - Незакрашенный кружок.
      • square - Закрашенный квадратик.
      • decima(l?) - Арабские цифры.
      • lower-roman - Строчные римские цифры.
      • upper-roman - Прописные римские цифры.
      • lower-alpha - Латинские строчные буквы.
      • upper-alpha - Латинские прописные буквы.
    • list-style-image - Картинки-маркеры.
    • list-style-position - Положение маркера
      (может быть выдвинут влево).
      Варианты:
      • inside - Маркер встраивается в список,
        и становится как бы первым символом.
      • outside - Маркер смещен влево.
  • ‹ul› - маркированный список
    Атрибуты: type со значениями:
    • circle - не закрашенные кружки
    • disc - закрашенные кружки
    • square - закрашенные квадраты
    • пример: ‹ultype=circle›
  • ‹li› - строка списка. Можно не закрывать.
    Маркеры картинки - вместо li вставляется img, без списка.
    Альтернатива описана двумя пунктами выше.
    Пример:
    ‹img src="marker.gif"› текст‹br›
    ‹img src="marker.gif"› текст‹br›
    ‹img src="marker.gif"› текст‹br›
    ‹img src="marker.gif"› текст‹br›
  • ‹dl›(контейнер) ‹dt›(Заголовок) ‹dd›(определение) - Списки определений.
  • ‹! DOCTYPE html› - тип документа
  • ‹title› - название документа
  • ‹base› - точный адрес(url) документа
  • ‹meta› - параметры, управляющие загрузкой и
    обработкой документа. Не имеет закрывающего тега.
    Атрибуты:
    • name - имя свойства
    • content - значение св-ва
    • scheme - имя схемы, используемой для обработки
      значения св-ва
    • http-equiv - используется вместо атрибута name,
      для указания имени http-сообщения
    • lang - информация о языке(не обязательно)
    • dir - направленность текста(не обязательно)
    • pragma - оператор META, запрещает кэширование док-та
  • ‹link› - подключает стили css. Эта
    информация может использоваться поисковыми машинами
    для оптимизации поиска. Задает заглавные ссылки.
    Примеры:
    • href="file.css" - Подключение стилей,
      находящихся в файле "file.css".
  • ‹style› - задает стили в самом html, имеет
    приоритет на css файлом. Атрибуты:
    type - обязательный атрибут, указывает язык таблицы стилей.
    (text/css или text/javascript)
    media - необяз. атрибут, задает целевое устройство для
    информации о стиле.
  • ‹script› - размещает код скриптов(javascript,
    VBScript, JScript). Может быть и в заголовке и в теле,
    и несколько раз.
    Атрибуты:
    • type - язык, на котором написан скрипт.
      (type="text/javascript")
    • src - месторасположение внешнего скрипта(src=URL)
    • language - язык скрипта в эл-те. НЕЖЕЛАТЕЛЬНЫЙ ЭЛ-Т.
      Вместо него исп-ся атрибут type
    • defer - необязательный. Сообщает браузеру, что скрипт
      не будет генерировать содержимое текста
    • charset - кодировка символов. Необязат.
  • ‹body›
    Атрибуты:
    • ВСЕ АТРИБУТЫ BODY НЕОБЯЗАТЕЛЬНЫ И НЕЖЕЛАТЕЛЬНЫ,
      Т.К. ИСПОЛЬЗУЕТСЯ ДЛЯ ЭТОГО CSS.
    • alink=color - цвет активной ссылки
    • link=color - цвет непросмотренной ссылки
    • vlink=color - цвет просмотренной ссылки
    • bgcolor=color - фон документа
    • background=URL - месторасположение фонового изображения
    • bgproperties - в положении fixed фон не прокручивается
      По умолчанию стоит nonfixed.
    • text=color - цвет документа
    • leftmargin, rightmargin, topmargin=pixels - размер полей
    • scroll - в положении no - полоса прокрутки док-та
      не отображается. По умолчанию стоит yes.
  • ‹i› - выделение текста курсивом
  • ‹b› - выделение текста жирным шрифтом
  • ‹abbr› - аббревиатура. С помощью атрибута
    title, можно сделать всплывающую подсказку к ней.
  • ‹acronim› - с тегом title, указывает расшифровку
    аббревиатуры. НЕ ПОДДЕРЖИВАЕТСЯ HTML5!!!
  • ‹cite› цитата(курсивом)
  • ‹code› - фрагмент компьютерного кода
  • ‹dfn› - обозначает текст как определение
  • ‹em› - курсив. Более предпочтителен чем ‹i›
  • ‹kbd› - обозначает текст, как
    введенный пользователем с клавиатуры
  • ‹mark› выделенный текст(по умолчанию, браузер
    отмечает его желтым)
  • ‹samp› - отмечает текст как пример.
  • ‹strong› жирный шрифт. Более предпочтителен
    чем ‹b›
  • ‹var› - экземпляр переменной или аргумента
    какой либо программы.
  • ‹del› и ‹ins› - ‹del›Отмечает старые удаленные
    данные    ‹/del›, и ‹ins›новые, вставленные вместо них‹/ins›.
    ‹del› является более предпочтительной альтернативой ‹s›
  • ‹pre› - отформатированный текст. Отображается
    браузером, в том виде, в котором он представлен в html-док-те.
    Например, рисунок из набора символов.
  • ‹q› - короткая цитата(без абзаца)
  • ‹blockquote› - Длинная цитата(блоком)
  • ‹p› Абзац, блок текста.
  • ‹br› - перенос строки. Атрибут clear смещает
    текст под картинку, прекращая его обтекание. Например:
    clear=left, для смещения под левую картинку.
  • ‹wbr› - Рекомендуемое место переноса строки,
    если поля блока не хватает.
  • ‹hr› - Линия. Атр-ты: align, size, width,
    noshade(отмена рельефности), color.
  • ‹details› - Скрывает текст.
    Пример скрытого текста.
  • ‹summary› - Задает заголовок для скрытого текста.
  • ‹menu› - Создает список меню. Элементы меню
    заключены в тег ‹li›, как и в списках.
    Атрибут type задает тип меню. Пока не поддерж-ся браузерами!
    Значения type
    • context - контекстное меню.
    • toolbar - меню в виде панели инструментов.
    • list - меню-список.
  • ‹table› - таблицы.
    Теги таблиц.
    • th - Table Header. Строка заголовка таблицы.
      Полужирный шрифт, выравнивание по центру.
    • tr - Table Row. Контейнер для каждой отдельной строки
      таблицы. Содержится внутри ‹table›, в себе
      содержит ‹th› или ‹td›.
    • td - Table Data. Обычная строка таблицы.
      Атрибуты для td и th:
      • abbr - Предоставляет сокращённую форму содержимого ячейки.
        Значение - текст в кавычках.
      • rowspan - Объединение соседних строк в столбце.
      • colspan - Объединение соседних столбцов в строке.
      • nowrap - Запрещает разбивать ячейку на несколько строк.
        НЕЖЕЛАТЕЛЬНЫЙ АТРИБУТ.
  • ‹caption› - Задает название таблицы. Находится внутри ‹table›.
  • ‹Группы строк›
    • ‹thead› - Шапка таблицы.
    • ‹tfoot› - Подвал таблицы. В документе html этот тег
      проводится до ‹tbody›, хотя отображается после.
    • ‹tbody› - Тело таблицы.
  • ‹colgroup› - Группирует по расположенным рядом столбцам.
    Первый ‹colgroup›(если перед ним нет col), Выделяет группу столбцов,
    начиная с нулевого. Последующие с первого столбца.
    Атрибуты:
    • span - Задает кол-во столбцов в группе. span=N
  • ‹col› - Группирует столбцы расположенные в разных местах таблицы.
  • ‹script› - Включает код скрипта в html-документ. Может быть
    расположен как в заголовке так и в теле документа.
    Как задать язык скриптов(обязательно!)
    1. Установить язык по умолчанию(в ‹meta›).
    2. Указывать отдельно для каждого скрипта(Будет приоритет над ‹meta›).
    СОБЫТИЯ(HTML5):
    • onclick - Однократное нажатие(курсор наведён).
    • ondblclick - Двойное нажатие(-/-/-/-).
    • onload - Происходит, когда браузер закончил загружать
      документ в окно или фрейм. Задается у BODY и FRAMESET.
    • onunload - onload - Происходит, когда браузер закончил удалять
      документ из окна или фрейма. Задается у BODY и FRAMESET.
    • onmousedown - Нажатие кнопки мыши(курсор наведён).
    • onmouseup - Отпускание кнопки мыши(курсор наведён).
    • onmouseover - Курсор находится над содержимым.
    • onmousemove - Перемещение курсора над содержимым.
    • onmouseout - Перемещение курсора за пределы содержимого.
    • onfocus - Происходит, когда элемент(для которого это установлено),
      получает фокус(мышь,tab)
      В каких тегах бывает:
      • label
      • input
      • select
      • ‹textarea›
      • button
    • onblur - Переход фокуса с элемента.
    • onkeypress - Нажатие и отпускание любой клавиши
      на эл-те. Определено почти для всех тегов.
    • onkeydown - Нажатие любой клавиши
      на эл-те. Определено почти для всех тегов.
    • onkeyup - Отпускание любой нажатой клавиши
      на эл-те. Определено почти для всех тегов.
    • onsubmit - Происходит при отправке формы.
      (Только FORM).
    • onreset - Происходит при сбросе содержимого
      формы. (Только FORM).
    • onselect - Происходит при выделении текстового
      фрагмента в текстовом поле. (теги INPUT и ‹textarea›)
    • onchange - Потеря фокуса, если с момента получения
      фокуса, содержимое было изменено. Теги заполнения
      форм (select, input, ‹textarea›)
  • ‹noscript› - Альтернативное отображение скрипта,

  • ‹canvas› - Задает область страницы, на которой
    с помощью JavaScript можно отобразить: рисованные объекты,

  • ‹a› - Контекстные ссылки.
    Атрибуты:
    • Допустимо не задавать якоря и не использовать атрибуты
      href, name, id, если это будет задано скриптами.
    • name - (Необязательный) Присваивает имя ‹a›
      Задает целевые якори(навигация по сайту с исп-ем id)
    • href - (Обязательный) Месторасположение целевого
      web-ресурса.
    • hreflang - (Необязательный) базовый язык целевого
      web-ресурса.
    • rel - Отношение текущего ресурса к тому, на который
      ссылается. Практически не исп-ся. Программирует клавишу

    • rev - Тоже что rel, но при переходе по обратной ссылке.
      "Назад".
    • type=content-type - Тип содержимого.
    • shape - Применяется в случае использования клиентских
      навигационных карт. Значение атрибута - форма активной
      области.
    • coords - Применяется в случае использования клиентских
      навигационных карт. Значение атрибута - граничные
      координаты активной области.
    • accesskey - Указываются "горячие клавиши" для
      активизации данной ссылки.
    • tabindex - Определяет последовательность перехода
      между ссылками при использовании TAB.
    • download - Скачивание документа по ссылке.
  • ‹img› - Изображение.
    Атрибуты:
    • align - Выравнивание
      • left - Выравнивание по левому краю.
        Текст обтекает по правой стороне.
      • right - Выравнивание по правому краю.
        Текст обтекает по левой стороне.
      • top - Выравнивание по верхней границе.
        Изображение не обтекается, а воспринимается
        как элемент строки.
        Верхняя граница изображения находится
        на одном уровне с верхней границей
        самого высокого элемента в текущей строке.
      • middle - Середина изображения находится
        на уровне базовой линии текущей строки.
      • bottom - Строка по низу картинки.
    • width и height - Размеры картинки.
      Преимущества использования:
      • Если не грузятся картинки, структура сохраняется.
        Блоки картинок остаются на своих местах.
      • Чтобы отрисовать например, линию, можно взять
        маленькую картинку и растянуть, не грузя сайт.
    • hspace и vspace - Отступы между текстом и изображением.
    • border - Рамка.
    • alt - Альтернативный текст.
  • ‹audio и video› - Вставка аудио и видео.
    Путь к файлу указывается тегом ‹source› или атрибутом src.
    Атр-ты audio
    • autoplay - Автоматическое воспроизведение.

    • loop - Повторное воспроизведение.
    • preload - Загрузка видеофайла вместе со страницей.
    • src - Путь к источнику видео.
    • height, width - Область воспроизведения видео.
  • ‹object› - Контейнер для вставки мультимедиа.
    Возможны три варианта обработки браузером:
    1. Браузер может определить тип содержимого и у него имеются
      все необходимые модули. Тогда он обрабатывает
      все значения ‹param›.
      Альтернативные варианты игнорируются.
    2. Браузер может определить тип содержимого но у него нет
      необходимого модуля. В этой ситуации браузер, с согласия пользователя
      пытается подгрузить этот модуль с адреса указанного атрибутом classid
      или в установках самого браузера. Если получается, то ситуация
      переходит в ↑первый↑ вариант. Если нет, то тег object и param внутри него
      - игнорируются. Отображается альтернативное содержимое object.
    3. Браузер изначально не может определить тип содержимого объекта.
      Тогда тег object со всеми param игнорируется полностью.
      Отображается альтернативное содержимое. В качестве альтернативного
      содержимого тега object может исп-ся другой тег object.
      При этом param альтернативного object игнорируются первичным тегом object.
    Атрибуты object:
    • data - URL-адрес файла, в котором содержится объект.
    • classid - Используется в двух вариантах:
      • Указание адреса модуля ActiveX, и позволяющего
        пользователю(если у него нет этого модуля) скачать его
        и установить у себя на компе.
      • Подключение имеющегося управляющего элемента ActiveX,
        указывая его индетификационное имя(шеснадцатеричный код).
    • codebase - Указывает базовый URL-адрес объекта,
      относительно которого будут браться все относительные ссылки.
      Если он отсутствует, по умолчанию в качестве базового адреса
      будет браться URL текущего документа.
    • codetype - тип содержимого данных, при загрузке codebase
      Рекомендуемый атрибут, т.к. позволяет браузеру не загружать
      неподдерживаемую инфу. Если отсутствует, по умолч-ю исп-ся значение type.
    • type - MIME-тип содержимого мультимедиа-файла.
      Настоятельно рекомендуется к использованию.
    • border - Задает толщину рамки(по умолч-ю её нет).
    • height - Вертикальный размер в пикселах.
    • width - Ширина в пикселах.
    • hspace - Горизонтальный отступ(в пикселах) между
      объектом и остальным содержимым док-та.
    • vspace - Вертикальный отступ(в пикселах) между
      объектом и остальным содержимым док-та.
    • align - Выравнивание объекта в документе.
      • left - Объект прикрепляется к левому краю док-та.
      • right - Объект прикрепляется к правому краю док-та.
      • top - Выравнивание по верхней границе
        текущей строки текста.
      • middle - Середина объекта на уровне базовой линии строки текста.
      • bottom - Нижняя граница объекта на базовой линии строки текста.
        Используется по умолчанию.
    • id - Задает имя объекта в пределах документа.
    • tabindex - Номер объекта в последовательности перехода
      клавишей TAB.
    • style - Стили.
    • usemap - Связывает объект с навигационной картой.
      Значение - имя навигационной карты.
    • onclick, ondblclick, onmousedown, onmouseup,
      onmouseover, onmousemove, onmouseout, onkeypress,
      onkeydown, onkeyup       - Внутренние события.
  • ‹embed› - Контейнер для вставки мультимедиа. Более предпочтителен.
    Более корректно поддерж-ся браузерами и более просто в использовании.
    Атрибуты:
    • src - URL-адрес мультимедиа-файла, в котором
      хранятся данные объекта.
    • type - MIME-тип содержимого мультимедиа-файла.
      Является необязательным, но настоятельно рекомендуется.
    • border - Толщина рамки в пикселах.
    • frameborder - Указывает наличие или отсутствие рамки
      вокруг объекта.
    • height - вертикальный размер в пикселах.
    • width - ширина в пикселах.
    • hspace - Размеры горизонтальных отступов
      между объектом и остальным содержимым документа,
      в пикселах.
    • vspace - Размеры вертикальных отступов
      между объектом и остальным содержимым документа,
      в пикселах.
    • hidden - Делает объект невидимым.
    • pluginspage - Указывает URL-адрес ресурса,
      в котором содержатся инструкции по инсталяции модуля.
    • align - Выравнивание. Значения как для object.
    • id - Имя объекта в пределах текущего документа.
  • ‹param› - задает список параметров и их значений
    для задания параметров обработки содержимого объекта.
    Конечный тег запрещен!
    Атрибуты:
    • name - имя параметра, понятное подключенному модулю.
    • value - Значение параметра, заданного name.
      Определяется подключенным модулем.
    • valuetype - тип атрибута value.
      • data - Означает, что указанное для атрибута value
        значение, будет определяться и передаваться модулю в виде строки. По умолчанию.
      • ref - Означает что значение value это URL-адрес файла
        в котором хранятся значения параметров.
      • object - Реализует ссылку на другой объект
        в этом же док-те. Значение - id другого объекта.
    • type - Тип содержимого(content-type) ресурса, чей URL указан
      атрибутом value. Исп-ся только если для valuetype
      указано значение ref.
    • id - Задает имя эл-та в пределах документа.
  • ‹figure› - Группировка объектов. Например текст и картинка.
  • ‹fig-caption›(или ‹figcaption›???) - Задает подпись к изображению.
  • ‹header› - Заголовок страницы.
  • ‹footer› - Подвал страницы.
  • ‹article› - Задаёт содержание страницы.
  • ‹section› - Разделы документа.
  • ‹aside› - Боковая панель или "сайдбар"(Меню, разделы сайта).
  • ‹main› - Основное содержимое документа.
  • ‹nav› - Навигация по сайту. Внутри располагаются ссылки.
  • ‹map› - Контейнер для навигационной карты(графическое меню со ссылками).
    Атрибуты:
    • name - Задает имя навигационной карте:
      ‹map name="Имя навигационной карты"›
    • usemap - Атрибут для ‹img›, или другого элемента.
      Устанавливает связь с навигационной картой.
      Пример: ‹img src="map.gif" usemap=#mymap›
  • ‹area› - Задает активную область(куда кликать).
    Один тег для одной ссылки.
    Атрибуты:
    • shape - Какая форма у активной области.
      Значения:
      • rect - Прямоугольник(по умолчанию).
      • circle - Круг.
      • poly - Многоугольник.
      • default - Всё изображение. Не все браузеры поддерживают,
        поэтому рекомендуется заменять его rect с размерами
        всего изображения.
    • coords - Размер и положение активной области на экране.
      Относительно верхнего левого угла. Варианты:
      1. Для прямоугольника: Значения указываются через запятую.
        Х и У-координаты для верхнего левого угла,
        Х и У-координаты для нижнего правого угла.
      2. Для круга: Через запятую, Х и У-координаты центра,
        и его радиус.
      3. Для многоугольника - Список координат вершин:
        Х1, У1, Х2, У2, Х3, У3 и т.д.
    • nohref - Указывает браузеру, что с активной областью
      не связана ни одна ссылка.
    • href - Указывает URL-адрес ссылки, прикрепленной
      к данной области.
      Если для активной области, задаваемой тегом area,
      не указаны оба атрибута(href и nohref),
      то по умолчанию браузер использует nohref.
    • target - Используется при работе с фреймами.
      Указывает имя фрейма, в котором должен отображаться
      документ, который открывается по ссылке, связанной
      с данной активной областью.
  • ‹iframe› - Рамка(блок), встраиваемая на сайт
    в определенное его место, внутри которой можно прокручивать
    информацию, и которая будет находиться на том же месте,
    при прокрутке основного сайта.
    Список атрибутов:
    • name - Имя фрейма.
    • width - Ширина встраиваемого фрейма.
    • height - Высота встраиваемого фрейма.
    • src - URL-адрес ресурса, загружаемого в фрейм.
    • align - Горизонтальное и вертикальное выравнивание
      фрейма в документе. С помощью него осуществляется привязка
      фрейма к краям документа, что делает возможным
      обтекание его текстом.
    • scrolling - Показывает браузеру о наличии
      у данного фрейма полосы прокрутки.
      Принимает значения:
      • yes - Полоса прокрутки отображается всегда.
      • no - Полоса прокрутки не отображается никогда.
      • auto - По необходимости(по умолчанию).
    • marginwidth - Отступ содержимого от правого и левого
      полей(в пикселах). Должно превышать 1 пиксел
      (задается браузером по умолч-ю).
    • marginheight - Отступ содержимого от верхнего и нижнего
      полей(в пикселах). Должно превышать 1 пиксел
      (задается браузером по умолч-ю).
    • frameborder - Вид границ вокруг фрейма.
      Возможные значения:
      • 1 - Объемные границы между текущим и
        прилегающими к нему фреймами. По умолч-ю.
      • 0 - Плоские линии определенного цвета и толщины.
  • ‹form› - Контейнер для формы
    (ввод данных для отправки на сервер).
    Он указывает адрес, куда будет послана форма,
    способ пересылки и характеристику данных,
    содержащихся в форме.
    Управляющие теги:
    • Кнопки - задаются тегами ‹button› и ‹input›.
      Различают:
      • Кнопки отправки - При нажатии отправляют форму на сервер.
      • Кнопки сброса - При нажатии устанавливают
        управляющие элементы в первоначальное положение.
      • Прочие кнопки - Кнопки, для которых не указано
        действие, выполняемое по умолчанию.
    • Радиокнопки - Задаются тегом input, и представляют собой
      переключатели "вкл/выкл". Если несколько кнопок заданы
      с одинаковыми именами, то их действия становятся взаимоисключающими.
      Например, если одна из них становится в положение "вкл",
      то все остальные автоматически встают в положение "выкл".
    • Флажки - Задаются тегом input, и представляют собой
      переключатели "вкл/выкл". В отличие от Радиокнопок,
      включенные флажки могут иметь одинаковые имена.
      Имена задаются атрибутом name.
    • Меню - Задается тегами ‹select›, ‹optgroup› и ‹option›.
      Представляет собой список возможных вариантов выбора.
    • Ввод текста - Реализуется тегами ‹input› - для одной строки,
      и тегами ‹textarea›, для нескольких строк.
    • Выбор файлов - Позволяет вместе с формой отправлять выбранные файлы.
      Реализуется тегом ‹input›.
    • Скрытые управляющие элементы - Создаются управляющим тегом ‹input›.
    Атрибуты тега form:
    • action - Обязательный атрибут! Его значение указывает
      URL-адрес запрашиваемой CGI-программы, которая будет
      обрабатывать данные, содержащиеся в форме.
      Допустимая запись - mailto:URL - фоорма будет послана
      по эл. почте. Если атрибут action не указан,
      содержимое формы будет отправлено на URL-адрес,
      с которого загружалась данная WEB-страница.
    • method - Метод HTTP, используемый для пересылки
      данных от браузера к серверу. Значения:
      • get - Данные пересылаются в первой строке
        HTTP-заголовка. Добавляются к URL,
        указанному action и пересылаются вместе с ним.
        Рекомендуется, если форма
        не вызывает никаких изменений на сервере. Используется по умолчанию.
      • post - Данные пересылаются серверу в тексте
        HTTP-сообщения.
        При использовании этого метода,
        допустимо написание только символов ASCCI.
    • enctype - Тип содержимого формы, используемый
      для определения формата кодирования, при её пересылке.
      Имеет три значения: application/x-www-form-urlencoded(умолч-е),
      multipart/form-data, text/plain.
    • id, class - Идентификаторы в пределах документа.
    • lang, dir - Информация о языке и направленности текста.
    • title - Заголовок эл-та.
      Выводится при наведении курсора на содержимое.
    • style - Встроенная информация о стиле.
    • onclick, ondblclick, onmousedown, onmouseup,
      onmouseover, onmousemove, onmouseout, onkeypress,
      onkeydown, onkeyup - onsubmit, onreset
      - Внутренние события.
  • ‹select› - Меню с выпадающим списком вариантов.
    Каждый вариант задается тегом ‹option›.
    Атрибуты:
    • name - Имя.
    • size - Число видимых элементов выбора.
    • multiple - Возможность выбора нескольких вариантов.
    • tabindex - Порядок выбора клавишей TAB.
    • disabled - Отключение элемента select.
    • onfocus, onblur, onchange - Внутренние события.
  • ‹option›
    Атрибуты:
    • value - Значение, которое пересылается серверу.
    • selected - Выбор по умолчанию.
    • disabled - Отключает ‹option›
    • id, class - Идентификаторы.
    • lang, dir - Язык и направленность текста.
    • title - Заголовок эл-та. Выводится при наведении курсора.
    • style - Стили.
    • onfocus, onblur, onchange, onclick, ondblclick,
      onmousedown, onmouseup, onmouseover, onmousemove,
      onmouseout, onkeypress, onkeydown, onkeyup
      - Внутренние события.
  • ‹textarea› - Многострочный ввод текста.
    Атрибуты:
    • name - Имя.
    • rows - Число видимых текстовых строк.
    • cols - Ширину в виде кол-ва символов.
    • id, class - Идентификаторы.
    • lang, dir - Язык, направленность текста.
    • title - Заголовок.
    • style - Стиль.
    • readonly - Не позволяет пользователю изменять поле.
    • tabindex - Порядок выбора клавишей TAB.
    • disabled - Отключает ‹textarea›
    • onfocus, onblur, onchange, onclick, ondblclick,
      onmousedown, onmouseup, onmouseover, onmousemove,
      onmouseout, onkeypress, onkeydown, onkeyup
      - Внутренние события.
  • ‹input› - Создает внутри формы поля:
    ввод текста, имя файла, пароль и др.
    Атрибуты:
    • name - Имя управляющего эл-та, образованного
      данным элементом input. Обязательный атрибут.
    • type - Обязательный атрибут., определяющий тип управляющего эл-та.
      Типы управляющего эл-та:
      • text - Создает элемент для ввода строки. Атрибуты ‹input› для управляющего элемента типа text(type=text):
        1. maxlength=n - Максимальное кол-во символов в строке
          (по умолчанию не ограничено).
        2. size=n - Количество отображаемых символов
          в строке ввода(в кол-ве символов).
        3. value=text - Начальное занчение текстового поля,
          используемое по умолчанию.
      • password - Аналогичен предыдущему. Разница в том, что отображаться будет
        не текст а звёздочки.
        (Обеспечивает очень слабую защиту!)
      • checkbox - Создает флажок on/off.
        Можно использовать несколько флажков
        с одинаковым name.
      • radio - Радиокнопка(с зависимой фиксацией).
        Используется в группах элементов с одинаковым именем,
        где только один элемент может быть "on"
        (остальные автоматически встанут в "off").
      • submit - Кнопка отправки.
      • image - Кнопка отправки в виде графического изображения.
      • reset - Кнопка сброса.
        По умолчанию с надписью "reset"
        (можно изменить в input значением value).
      • file - Элемент выбора файла.
        Для передачи этих файлов,
        при задании формы нужно в атрибутах указать:
        enctype="multipart/form-data" и method=post
      • hidden - Создает скрытый элемент,
        не отображаемый браузером.
        Содержащаяся в нём информация пересылается серверу
        при отправке формы и не может быть изменена
        ни браузером ни пользователем.
      Доп. атрибуты input при type=checkbox:
      • value=text - Обязательный атрибут для флажков.
      • checked - Необязательный атрибут,
        который устанавливает флажки на "on".
        Если он не указан, то флажки будут стоять в "off".
      Доп. атрибуты input для радиокнопок type=radio:
      • value=text - Обязательный атрибут для флажков.
      • checked - Необязательный атрибут,
        который устанавливает флажки на "on".
        Если он не указан, то флажки будут стоять в "off".
      Атрибуты input для кнопок отправки type=submit:
      • value=text - Изменяет надпись на кнопке.
      Атрибуты input для графического
      изображения кнопок отправки:
      • src - URL-адрес изображения.
      • align - Тип выравнивания изображения.
      • alt - Рекомендуемое пояснение, на случай
        не отображения картинки.
    • size - Начальная ширина управляющего элемента(px).
      Для элементов типа text и password -
      кол-во символов в строке.
    • align - Выравнивание.
    • id, class - Идентификаторы.
    • lang, dir - Язык, направленность текста.
    • title - Заголовок эл-та.
    • style - Стили.
    • alt - Краткая поясняющая инф-я.
    • usemap - имя клиентской навигационной карты.
    • readonly - Запрет на изменение поля.
    • tabindex - Порядок выбора клавишей TAB.
    • disabled - Отключает данный элемент формы.
    • accesskey - "Горячие клавиши" доступа
      к данному управляющему элементу.
    • onfocus, onblur, onselect, onchange, onclick, ondblclick,
      onmousedown, onmouseup, onmouseover, onmousemove,
      onmouseout, onkeypress, onkeydown, onkeyup
      - Внутренние события.
  • ‹button› - Кнопки, используемые для различных целей.
    Обычно исп-ся с атрибутом onclick, в значении которого
    прописывается сценарий, при нажатии на кнопку.
    Используется для создания красочного пользовательского интерфейса.
    Атрибуты:
    • name - Имя управляющего эл-та.
    • value - Значение, которое будет передано
      сценарию(если задан name).
    • type - Тип кнопки button.
      Варианты:
      • submit - Кнопка отправки, аналогичная
        кнопке в input.(по умолч-ю)
      • reset - Кнопка сброса, аналогичная
        кнопке в input.
      • button - Уникальная пользовательская кнопка,
        функции которой задает сам разработчик.
        Именно здесь исп-ся атрибут onclick.
    • disabled - Отключает кнопку.
    • accesskey - "Горячие клавиши" для быстрого доступа к кнопке.
    • usemap - Связывает кнопку с навигационной картой.
      Значение должно совпадать с атрибутом name,
      тега ‹map›, т.к. между ними устанавливается связь.
    • tabindex - Порядок выбора клавишей TAB.
    • onclick, ondblclick, onmousedown, onmouseup,
      onmouseover, onmousemove, onmouseout, onkeypress,
      onkeydown, onkeyup - onsubmit, onreset
      - Внутренние события, для обработки которых
      указывается имя соответствующего сценария.
  • ‹label› - Зона, расположенная рядом с флажком/радиокнопкой/submit,
    при клике на которую срабатывает действие.
    Атрибуты:
    • for - Обязательный атрибут.
      Связывает метку с управляющим элементом.
    • id, class - Идентификаторы.
    • lang, dir - Язык, направленность текста.
    • title - Заголовок эл-та.
    • style - Стили.
    • accesskey - "Горячие клавиши" для быстрого доступа к кнопке.
    • tabindex - Порядок выбора клавишей TAB.
    • onclick, ondblclick, onmousedown, onmouseup,

      onkeydown, onkeyup - onsubmit, onreset
      - Внутренние события, для обработки которых

  • ‹fieldset› - Группировка связанных меток

  • ‹legend› - Задаёт заголовок группе, выделенной ‹fieldset›.
    Атрибуты:

    • style - Стили.
    • accesskey - "Горячие клавиши" для быстрого доступа к кнопке.
    • onclick, ondblclick, onmousedown, onmouseup,

      - Внутренние события, для обработки которых
